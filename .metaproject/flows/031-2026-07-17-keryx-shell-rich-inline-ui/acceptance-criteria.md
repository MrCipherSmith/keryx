# Acceptance Criteria

Rules:

- Criteria lines use the exact format `- ACn: <criterion>`.
- After `flow freeze` this file is checksum-protected: any edit outside
  `keryx flow ac update` fails every gate and status transition.
- Completion requires every ACn to be confirmed via
  `keryx flow ac confirm <id> <ACn>`.

## Criteria

- AC1: Additive IO seam (backward compatible) — `ShellIO` (`src/commands/shell.ts`) gains OPTIONAL `onTurnStart?(): void`, `onTurnEnd?(full: string): void`, and `onSystem?(text: string): void`. `runShell` calls `onTurnStart` immediately before the provider stream loop, `onTurnEnd(accumulated)` after any turn that produced assistant content, and routes every NON-token write (error, `/help`, `/connect`, unknown-command, "not available", and the anthropic-no-key notice) through a `system(t) = io.onSystem?.(t) ?? io.write(t)` helper; assistant token deltas still go through `io.write`. When none of the optional hooks are supplied, `runShell` output is byte-identical to the pre-change behavior — proven by a regression test that runs the existing hook-less `{lines, write}` IO and asserts the exact same accumulated `write` output as before.
- AC2: Pure markdown renderer — `src/lib/ui.ts` exports `renderMarkdown(md: string): string` that styles ATX headings (`#`..`######`), `**bold**`, `` `inline code` ``, fenced ```` ``` ```` code blocks, and `-`/`*` bullet lists using the existing `style`/`symbols`/`colorEnabled()` helpers, and returns the input as plain text (no escape codes) when `colorEnabled()` is false. It is a deterministic pure `string→string` function (no `Date.now`/`Math.random`, no IO) with unit tests in `src/lib/ui.test.ts` covering each element under BOTH `FORCE_COLOR=1` (emits ANSI) and `NO_COLOR=1` (plain, structurally faithful) — including a fenced code block and a multi-item list.
- AC3: Rich TTY rendering (wrapper) — `shellCommand` renders a styled session header (provider/model), a colored input prompt with a role marker, an assistant role label + animated spinner on `onTurnStart` that is cleared on the first streamed token, live raw token streaming during the turn, and a `renderMarkdown` re-render of the completed reply on `onTurnEnd`; `onSystem` styles errors in red and other notices dimmed. All styling is confined to the TTY wrapper (spinner timer included — the `runShell` core stays free of `Date.now`/`Math.random`/timers) and degrades to plain, uncorrupted output when `NO_COLOR` is set or `stdout.isTTY` is false. The wrapper remains not-unit-tested per existing convention (its behavior is exercised only by the manual live smoke).
- AC4: Reuse / scope / no-deps — the change is composition-only over flows 021/022: the only edits are additive (`src/commands/shell.ts` seam + wiring, `src/lib/ui.ts` renderer, `src/lib/ui.test.ts` tests, and the `shellCommand` wrapper); provider/model detection & selection (`src/commands/select.ts`), the provider adapters, the SSRF/loopback egress posture, and the JSONL-RPC runtime contract are reused UNCHANGED. No new production dependency is added — `dependencies` REMAINS `{}` (no Ink/OpenTUI/chalk/markdown library); hand-rolled ANSI via `src/lib/ui.ts` only. No full-screen TUI (alt-screen/scrollback/status-bar) is introduced. The frozen requirements package, canonical schemas, ADRs, `src/eval/`, and `src/contracts/` are NOT modified. The shell/CLI never write flow.json.
- AC5: No regression / offline / deterministic — `tsc --noEmit` is clean and the full `bun test` suite is ≥ the pre-change baseline of 1355 pass / 3 skip / 0 fail with the new tests green and 0 fail, and the ENTIRE automated suite runs OFFLINE (injected IO + FakeProvider; no real TTY, no live network). A live manual smoke of `keryx shell` (run via `bun src/cli.ts shell`) against local Ollama streams a real multi-turn reply showing the styled header, spinner, live streaming, and markdown re-render, with color correctly suppressed under `NO_COLOR=1` — recorded in the flow journal; not a CI gate.
