# Acceptance Criteria

Rules:

- Criteria lines use the exact format `- ACn: <criterion>`.
- After `flow freeze` this file is checksum-protected: any edit outside
  `keryx flow ac update` fails every gate and status transition.
- Completion requires every ACn to be confirmed via
  `keryx flow ac confirm <id> <ACn>`.

## Criteria

- AC1: SSRF finding closed (security) — `src/commands/select.ts` provider detection (`probeOllamaModels`) applies the reused W15 egress guard (`isPrivateEgressHost`/`isLoopbackHost` from `src/harness/mutation/guard.ts`) BEFORE issuing its fetch: a private / link-local / cloud-metadata `--base-url` host (e.g. `169.254.169.254`, `10.0.0.5`) causes ollama to be OMITTED from detection with NO fetch issued to that host (an injected-fetch spy shows 0 calls), while a loopback host (`localhost`/`127.0.0.1`) is still probed and offered — matching the guard the `OllamaProvider` chat path already enforces for the same host.
- AC2: Contained-process exit code surfaced (logic) — the `completed` variant of `ContainedProcessOutcome` carries `exitCode`, and `runContainedProcess` propagates `observation.exitCode` onto it (a clean-exit reporting exit 0 → `{kind:"completed", exitCode:0}`, exit 2 → `{kind:"completed", exitCode:2}`); the `completed` classification itself is unchanged (SC_R04 containment semantics) and every bound hit (timeout / output-overflow / cancelled / spawn-error) remains a recorded NON-success (never `completed`).
- AC3: Signal classification corrected (logic) — the real adapter classifies as `deadline-exceeded` ONLY a genuine deadline (`spawnSync` `ETIMEDOUT`, or a `SIGKILL` with the deadline actually reached); any other signal-terminated child (e.g. `SIGSEGV`, external `SIGTERM`) is classified as a distinct crash / non-success observation, NEVER `deadline-exceeded` and NEVER `completed`. The corrected logic lives in a PURE, side-effect-free classifier that is unit-tested OFFLINE (imported without constructing the adapter or spawning), and the real adapter stays capability-gated (`allowRealSubprocess`/`KERYX_ALLOW_REAL_SUBPROCESS`) and excluded from CI.
- AC4: Executor branches pinned (testing) — `executor.test.ts` now covers the two previously-unpinned branches with mutation-proof cases: (a) an external cancel (`input.cancelled===true`) overriding a CLEAN observation → `{kind:"cancelled"}` (the test fails if the `|| input.cancelled` clause is removed); (b) a clean observation whose `outputBytes` exceeds `outputLimitBytes` → `{kind:"output-overflow"}` (the test fails if the executor's finer-limit reclassification branch is removed). The stale `input.cancelled`-forwarded-to-spawn design-note is corrected.
- AC5: dispatchExtension dedup + no regression / scope — `src/harness/extension/execute.ts` builds the dispatch and result extensions through a single shared `buildExt(canonicalContract)` helper (the two differ only in `canonicalContract`) with NO behavior change (existing round-trip / transport-parity tests stay green); `tsc --noEmit` is clean and the full `bun test` suite is ≥ the baseline (1306 pass / 2 skip) with the new tests green and 0 fail; behavior is deterministic (injected id/clock; no `Date.now`/`Math.random` in the offline cores); no new production dependency (`dependencies` `{}`), no real spawn in the offline suite; the executor/adapter/extension write no flow.json (D-02); secrets (env values / API key) are never logged; the reused primitives (`guardAction`/egress predicates, `inheritBudget`, `ExecutionReceipt`, W7 evidence, `planWaves`) and the frozen requirements package, canonical schemas, `src/eval/`, `src/contracts/`, ADR-0001…0004 are NOT modified (edits are additive-only where needed). The deferred LOW/INFO items are out of scope.
