## Overview

`src/agents` owns the global agent-bootstrap lifecycle: it installs, checks, and removes a managed Metaproject routing block inside each supported AI coding agent's global instruction file (e.g. `~/.claude/CLAUDE.md`, `~/.config/opencode/AGENTS.md`). The module knows about five runtimes — Claude Code, OpenCode, ZCode, Codex, and Antigravity — and uses HTML comment sentinels to demarcate the block it controls so it can upsert or remove it without disturbing user-authored content. It is the only place in the codebase that writes to agent entrypoint files on disk.

## How it works

The module has two layers. The lower layer is a registry of `AgentBootstrapRuntime` descriptors — immutable value objects that capture an agent runtime's id, canonical file name (`AGENTS.md` or `CLAUDE.md`), path resolver (a function from a home root to the absolute file path), and a list of string aliases that the CLI can accept as synonyms. The upper layer is a set of async operations (`agentBootstrapStatus`, `installAgentBootstrap`, `uninstallAgentBootstrap`) that read and write those files using the block-manipulation helpers defined in the same file. Block manipulation is done through three private functions: `extractManagedBlock` locates the sentinel-delimited region in a string; `removeManagedBlock` strips it out and collapses excess blank lines; `upsertManagedBlock` removes any existing block then re-inserts a fresh one near the top of the file — after any YAML front-matter and after any leading `#` heading — via `insertBlockNearTop`. The actual block text (the Keryx Metaproject bootstrap instructions) is generated by `renderAgentBootstrapBlock`, which delegates to `renderGlobalMetaprojectBootstrapBlock` in `src/lib/agent-entrypoint-blocks.ts`. All async I/O uses Node's `fs/promises`; directory creation is handled with `mkdir({ recursive: true })` so a missing `~/.claude` directory is created on first install.

## Key concepts

- **AgentBootstrapRuntime** — a descriptor for one supported AI agent runtime: its stable `id`, human `label`, list of accepted CLI `aliases`, the `fileName` it uses (`AGENTS.md` or `CLAUDE.md`), and a `filePath(homeRoot)` resolver that returns the absolute path to that agent's global instruction file.
- **Managed block** — the region of an agent instruction file delimited by the HTML comment sentinels `<!-- keryx:global-bootstrap -->` and `<!-- /keryx:global-bootstrap -->`. Keryx owns only this region; everything outside it is user content.
- **AgentBootstrapStatus** — the read-only view of an agent file: whether the file exists, whether the sentinel is present (`installed`), and whether the installed block exactly matches what the current version of keryx would render (`current`). The `current` flag is how callers detect stale bootstrap blocks.
- **Install / uninstall result** — `AgentBootstrapInstallResult` and `AgentBootstrapUninstallResult` extend `AgentBootstrapStatus` with `wrote`/`removed` flags and a `dryRun` flag; when `dryRun` is true, status is computed from the would-be content rather than re-read from disk.

## Main flows

**Install bootstrap into an agent's instruction file**
`installAgentBootstrap(runtime, options)` reads the current file content (defaulting to a bare heading if the file does not exist), calls `upsertManagedBlock` to produce the updated string, and writes it back only when the content has changed and `dryRun` is false. It then re-reads the file (or computes status from the new content in dry-run mode) and returns `AgentBootstrapInstallResult` with `wrote: true/false`.

**Check whether the bootstrap block is up to date**
`agentBootstrapStatus(runtime)` reads the file, renders the expected block via `renderAgentBootstrapBlock`, and compares the extracted managed region against the expected text. It returns `installed: true` when the sentinel is present and `current: true` only when the installed block matches byte-for-byte. This lets `src/commands` surface a warning when the user's agent file has a stale block from an older keryx version.

**Resolve runtime identifiers from CLI input**
`resolveAgentBootstrapRuntimes(ids)` accepts an array of strings (e.g. `["claude", "opencode"]`, `["all"]`, or `[]`), expands `"all"` to the full runtime list, resolves aliases, deduplicates, and returns `{ runtimes, unknown }`. The `unknown` array lets the caller report unrecognised names without crashing.
